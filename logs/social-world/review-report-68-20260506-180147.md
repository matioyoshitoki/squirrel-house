## 严重问题（Major）

### 1. `HomePage` 冷启动已登录用户 `MatchingSocketCubit.connect()` 不触发（上轮遗留，仍未修复）
- **位置**: `apps/mobile/lib/presentation/pages/home_page.dart:51-56`
- **问题**: `BlocListener<AuthBloc, AuthState>` 仍使用了 `listenWhen: (prev, curr) => curr is AuthAuthenticated && prev is! AuthAuthenticated`。`BlocListener` 的 `listener` **仅在状态流发出新事件（即状态发生变化）时触发**，不会在 widget 初始挂载时触发。当已登录用户冷启动 App 时，`AuthBloc` 在 `HomePage` 构建前即已完成 `AuthCheckRequested` 并处于 `AuthAuthenticated` 状态，`HomePage` 挂载后不会再收到从非认证到认证的流转，导致 `MatchingSocketCubit.connect()` 永远不会被调用，实时匹配 Socket 断开。
- **影响**: 已登录用户冷启动 App 后无法接收实时匹配成功通知；仅在从登录页新登录（状态发生 `AuthUnauthenticated` → `AuthAuthenticated` 变化）时才能正常连接。
- **修复**: 去掉 `listenWhen`（`connect()` 已实现幂等，相同 token 重复调用无害），改为在 `listener` 内部判断 `state is AuthAuthenticated`：
  ```dart
  BlocListener<AuthBloc, AuthState>(
    listener: (context, state) {
      if (state is AuthAuthenticated) {
        context.read<socket.MatchingSocketCubit>().connect();
      }
    },
    child: ...,
  )
  ```
或者在 `initState` 中同步判断当前 Auth 状态并调用 `connect()`。
- **依据**: 上一轮 review report `review-report-68-20260506-173314.md` Major #1；`flutter.md` BLoC 注册层级规范（Socket 长连接必须在 App 级别管理且确保生命周期正确）；`MatchingSocketCubit.connect()` 的幂等实现（`matching_socket_cubit.dart:39-43`）。
