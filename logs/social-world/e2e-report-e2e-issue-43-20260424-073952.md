## 发现的问题与修复

### 1. 产品 Bug：Matching Gateway 启动崩溃
- **位置**: `apps/api/src/modules/matching/matching.gateway.ts:39`
- **现象**: API 启动后立即崩溃 `TypeError: Cannot read properties of undefined (reading 'opts')`，导致 `nest start --watch` 无限重启，E2E 登录请求全部失败。
- **根因**: `afterInit` 中直接访问 `sioServer.engine.opts.cors`，但 `engine` 或 `opts` 可能为 `undefined`。
- **修复**: 添加可选链保护：
  ```typescript
  if (sioServer.engine?.opts) {
    sioServer.engine.opts.cors = { origin: clientUrl };
  }
  ```
- **验证**: 修复后重启 API，`curl /auth/login` 返回正常，Redis SMS Code 不再被异常进程误删。

### 2. Flow 过时：`auth/login.yaml` 语法错误
- **位置**: `.maestro/flows/auth/login.yaml`
- **现象**: `Unrecognized field "optional"` 解析失败。
- **修复**: 将 scalar 形式的 `assertVisible: "完善资料"` + 同级 `optional: true` 改为 map 形式：
  ```yaml
  - assertVisible:
      text: "完善资料"
      optional: true
  ```

### 3. 环境问题：模拟器冷启动超时
- **位置**: `.maestro/includes/login-as-test-user.yaml`
- **现象**: `clearState: true` 后 Flutter App 首帧渲染耗时可达 20+ 秒（ProfileInstaller + Sentry 初始化），导致 `assertVisible` 在 10s 默认超时内失败。
- **修复**: 将首屏等待改为 `extendedWaitUntil`，超时提升至 30 秒：
  ```yaml
  - extendedWaitUntil:
      visible: "欢迎使用 Social World"
      timeout: 30000
  ```
- **验证**: 6 个 flow 单流运行时全部通过。

### 4. 脚本缺陷：`run-e2e.sh` 后端检测失效
- **位置**: `.maestro/scripts/run-e2e.sh`
- **现象**: `curl -sf http://localhost:3001` 返回 404，脚本误判后端未启动，仅运行 login flow。
- **修复**: 将健康检查改为探测实际存在的 API 端点 `/api/v1/auth/login` 的 HTTP 响应码。

### 5. Flow 不稳定：`swipe_recommendations.yaml` 文字断言
- **位置**: `.maestro/flows/matching/swipe_recommendations.yaml`
- **现象**: `assertVisible: "附近"` 失败，但 `assertVisible: id: mode_tab_nearby` 通过。
- **根因**: Flutter `Semantics(identifier: ..., label: ...)` 包裹 `Text` 后，Maestro 的文字匹配行为不一致。
- **修复**: 移除冗余的文字断言，保留更稳定的 `id` 断言。

## 批量运行说明

当使用 `maestro test flows/*.yaml` 一次性串联运行全部 flow 时，由于每个 flow 都携带 `clearState: true`，模拟器连续冷启动偶发出现 20s+ 的延迟，导致部分 flow 在 30s 等待后仍超时失败。**建议 CI 中采用分步运行或保留一次登录态后复用 token。**
