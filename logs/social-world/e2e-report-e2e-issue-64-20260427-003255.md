## 发现的问题与修复

### 1. 产品 Bug: `/conversations` API 返回 500
- **根因分析**: `apps/api/src/modules/im/im.service.ts` 的 `getConversations` 方法中，TypeORM QueryBuilder 对 `conversation_participants` 表使用了 `.innerJoin("conversations", ...)` 跨表关联，并使用 `.take()` 做分页。TypeORM 的 `.take()` 会生成 `DISTINCT` 子查询包裹层，但包裹层的 `SELECT` 列表不会包含被 `ORDER BY` 引用的关联表字段（`c.updated_at`），导致 MySQL 报错：`Unknown column 'distinctAlias.c_updated_at' in 'field list'`。
- **修复**: 将 `.take(parsedLimit + 1)` 改为 `.limit(parsedLimit + 1)`。由于 `conversation_participants` 对 `(conversation_id, user_id)` 有唯一索引，且查询已按 `user_id` 过滤，结果天然无重复，无需 `DISTINCT` 分页行为。
- **影响文件**:
  - `apps/api/src/modules/im/im.service.ts`
  - `apps/api/src/modules/im/tests/im.service.spec.ts`（同步更新 mock）
  - `tests/bdd/step-definitions/im_conversation_list.steps.js`（同步更新 mock）

### 2. Flow 过时: IM 流程未处理空会话列表
- **根因分析**: `enter_chat.yaml` 和 `send_message.yaml` 无条件尝试点击 `conversation_item`，但测试用户可能没有匹配记录/会话，导致元素找不到而失败。
- **修复**: 使用 Maestro 的 `runFlow` + `when: visible:` 条件，仅在 `conversation_item` 存在时执行聊天交互；否则断言空状态文案（"还没有匹配成功的好友" / "继续滑动，发现感兴趣的人吧"）。
- **影响文件**:
  - `.maestro/flows/im/enter_chat.yaml`
  - `.maestro/flows/im/send_message.yaml`

### 3. Flow 过时: Profile Setup 流程未处理已完善资料用户
- **根因分析**: `profile/setup.yaml` 强制断言 "完善资料" 页面可见，但 `clearState: true` 仅清除本地 Token，后端数据库中测试用户资料已完整，登录后直接跳转首页。
- **修复**: 将完善资料步骤提取到新的 include 文件 `profile-setup-actions.yaml`，在 `setup.yaml` 中通过 `runFlow` + `when: visible: "完善资料"` 条件执行；若资料已完善则断言首页 `tab_home` 可见。
- **影响文件**:
  - `.maestro/flows/profile/setup.yaml`
  - `.maestro/includes/profile-setup-actions.yaml`（新增）
