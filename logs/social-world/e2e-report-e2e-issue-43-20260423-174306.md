### ❌ Flow: profile/setup
- **结果**: FAILED (18s)
- **失败原因**: `Assertion is false: "完善资料" is visible`
- **根因分析**: **环境问题 / 产品功能不完整**
  1. `setup.yaml` 在 `launchApp` 后直接断言 "完善资料" 可见，期望应用已自动登录。
  2. 但失败截图显示应用实际停留在登录页（"欢迎使用 Social World"、手机号/验证码输入框、登录按钮均可见）。
  3. 从 `commands-(setup).json` 的 view hierarchy 确认，当前页面为 `/login`，无任何 "完善资料" 元素。
  4. 根本原因是 **登录流程实际上未成功**：
     - 前端 `_onSendCode` 在 MVP 阶段仅启动本地倒计时，**不调用后端 API**（`login_page.dart` 注释已说明）。
     - 后端 `AuthService.login()` 严格校验 Redis 中的 `sms_code:{phone}`，不存在则返回 `1001 Invalid or expired SMS code`。
     - 当前 E2E 环境（Redis）中未预置测试手机号 `13800138000` 的验证码，导致登录 API 调用失败，`AuthBloc` 进入 `AuthFailure` 状态，**不会保存 accessToken**。
     - 当 `setup` flow 重新 `launchApp` 时，`AuthCheckRequested` 读取不到 token，应用路由停留在 `/login`。
- **修复建议**: 
  - **环境层面**: 在 E2E 环境启动脚本中增加一步，预置测试验证码到 Redis：
    ```bash
    redis-cli SETEX sms_code:13800138000 300 123456
    ```
    或暴露 `AuthService.seedSmsCode()` 为内部 HTTP 端点，供 E2E 初始化调用。
  - **Flow 层面**: 当前 `login.yaml` 虽然通过，但缺乏对登录成功的断言。建议在 `login.yaml` 中增加登录成功后的页面断言（如 `assertVisible: "完善资料"` 或 `assertVisible: "首页"`），以便更早暴露登录失败问题。
- **步骤详情**:
  - launchApp ✅
  - assertVisible: "完善资料" ❌ (FAILED after 17s)
