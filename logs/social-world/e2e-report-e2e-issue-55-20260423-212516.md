## Flow 详情

### ✅ Flow: auth/login
- **结果**: PASSED (部分运行通过，但存在间歇性失败)
- **说明**: 
  - 该 flow 为 smoke 测试，仅验证登录页 UI 交互（输入手机号、验证码、点击按钮）。
  - **设计缺陷**：flow 在 `runFlow: includes/login-as-test-user.yaml` 后没有任何 `assertVisible` 断言验证登录结果。即使后端登录失败、App 停留在登录页，Maestro 也会判定通过。

### ❌ Flow: profile/setup
- **结果**: FAILED
- **失败原因**: 
  1. **间歇性白屏**（`clearState: true` 后 App 启动显示全白屏幕，持续 17s+ 不渲染）。
  2. 当白屏未触发时，flow 能完成所有 UI 交互步骤，但由于缺少对保存结果的验证，无法发现后端是否真正保存了资料。
- **根因分析**: 
  - **环境问题**：`launchApp: clearState: true` 在 Emulator 上间歇性导致 Flutter App 白屏启动，属于环境/模拟器不稳定问题。
  - **Flow 过时**：原 flow 缺少 `runFlow: includes/login-as-test-user.yaml` 步骤，已在本次修复。
  - **后端环境问题**：运行中的 backend (`localhost:3001`) 实际来自另一个项目目录 (`/Users/insulate/Desktop/social-world/`)，与当前 PR 代码不一致。该后端在每次登录后会删除 Redis 中的 `sms_code:13800138000`，导致需要登录的 flow 在重新运行时验证码失效。
- **修复建议**: 
  - 已修复：在 `profile/setup.yaml` 中补充了 `runFlow: includes/login-as-test-user.yaml`。
  - 已修复：在 `tapOn: "完成"` 后增加 `assertVisible: "首页"` 以验证资料保存成功并跳转首页。
  - 建议 CI 环境在跑 `profile/setup.yaml` 前预置 SMS Code 到 Redis，或确保 backend 保留 E2E 测试手机的验证码。
  - 建议调研 `clearState: true` 白屏问题，可尝试在 `launchApp` 后增加 `extendedWaitUntil` 等待 `"欢迎使用 Social World"` 出现（已部分尝试，但白屏时间可能超过 20s）。

## 环境问题详细说明

### 1. Backend 代码与项目目录不一致
- 端口 3001 上的 API 进程 `cwd` 为 `/Users/insulate/Desktop/social-world/apps/api`，而非当前测试项目 `/var/folders/lt/w3sl6z8j5v50vt2s6knvf5n00000gn/T/rework-pr-55-1814738584/apps/api`。
- 该 backend 的 `auth.service.ts` 在 `login()` 中没有保留 `sms_code:13800138000` 的逻辑（或版本不同），导致每次登录后 Redis 中的验证码被删除。
- **影响**：所有需要登录的 flow 在顺序/重复执行时，验证码会在第一次登录后失效。

### 2. `clearState: true` 导致 Flutter App 白屏
- 在多次测试中发现，`launchApp: clearState: true` 有约 50% 概率导致 App 启动后显示全白屏幕超过 17 秒（Maestro `assertVisible` 超时失败）。
- 该问题在 `login.yaml`、`profile/setup.yaml`、`swipe_recommendations.yaml` 均出现。
- 可能原因：Emulator 性能、`clearState` 与 Flutter Engine 初始化冲突、或 Maestro instrumentation 启动时序问题。

### 3. 测试用户数据未隔离
- 测试手机号 `13800138000` 在数据库中已存在，且 `user_profiles.is_complete = false`。
- 由于 `profile/setup.yaml` 从未真正跑通并保存成功，该用户一直处于资料不完整状态，导致所有依赖完整资料的 matching flow 无法进入首页。
