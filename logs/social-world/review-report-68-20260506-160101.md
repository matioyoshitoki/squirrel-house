## 严重问题（Major）

### 1. `LoginState.copyWith` 未正确保留 `validationError`
- **位置**: `apps/mobile/lib/presentation/blocs/login/login_state.dart:26`
- **问题**: `copyWith` 中 `validationError` 字段直接赋值 `validationError`（传入参数），而非使用 `?? this.validationError` 回退到当前值。当调用方仅更新其他字段（如 `copyWith(countdown: next)`）时，`validationError` 参数默认值为 `null`，导致当前 `validationError` 被意外重置为 `null`。
- **影响**: 
  - `LoginCubit` 的倒计时 `Timer.periodic` 每秒调用 `emit(state.copyWith(countdown: next))`，会将用户刚刚触发的表单验证错误在最多 1 秒后自动清除。
  - 破坏 `copyWith` 的"更新指定字段、保留其余字段"的基本语义，后续若将 `validationError` 用于输入框下方持续展示的错误文本，会出现闪烁或无法展示的问题。
- **修复**: 
  ```dart
  // 方案 A（推荐）: 使用 Object? 区分「未传」和「传 null」
  LoginState copyWith({
    bool? isCountingDown,
    int? countdown,
    bool? isLoading,
    Object? validationError = const _Sentinel(),
  }) {
    return LoginState(
      isCountingDown: isCountingDown ?? this.isCountingDown,
      countdown: countdown ?? this.countdown,
      isLoading: isLoading ?? this.isLoading,
      validationError: validationError is _Sentinel
          ? this.validationError
          : validationError as String?,
    );
  }
  // 并在 login_cubit.dart 中：
  // validationErrorShown() => emit(state.copyWith(validationError: null))
  ```
或方案 B: 增加显式 `bool clearValidationError = false` 参数。
- **依据**: `login_cubit.dart:18-25` 的 `Timer.periodic` 回调与 `login_state.dart:16-27` 的 `copyWith` 实现。
