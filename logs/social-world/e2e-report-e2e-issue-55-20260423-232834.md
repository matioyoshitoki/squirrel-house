### ❌ Flow: matching/swipe_recommendations
- **结果**: FAILED (34s)
- **失败原因**: `Assertion is false: id: mode_tab_nearby is visible`
- **根因分析**: **产品 Bug（Flutter BlocBuilder + Maestro 兼容性）**
  - 本 flow 使用 `clearState: true` 后通过 `runFlow: includes/login-as-test-user.yaml` 登录。
  - `login-as-test-user.yaml` 中 `tapOn: "登录"` 步骤因 `LoginPage` 将 `SwButton` 包裹在 `BlocBuilder<AuthBloc, AuthState>` 中，点击后立即触发 `AuthAuthenticating` → Widget 重建 → Maestro 的 accessibility tap 手势被 Flutter 框架取消，`onPressed` 回调从未执行。
  - 验证结果：
    - Maestro `tapOn` / `longPressOn` / `point` 均无法触发登录按钮。
    - 仅 `adb shell input tap 540 1145` 能正常触发并发起 API 请求。
  - 由于登录未实际完成，App 停留在登录页，`mode_tab_nearby` 自然不可见，断言失败。
- **修复建议**: 
  - **产品侧（推荐）**: 将 `LoginPage` 中的 `BlocBuilder` 替换为 `BlocListener` 处理导航，按钮本身保持静态，避免点击时重建。
  - **Flow 侧（临时）**: 需配合 ADB fallback 或修改 `run-e2e.sh` 在登录步骤插入 `adb shell input tap`。

## 额外发现

### 1. SMS 验证码有效期过短
- 后端 `AuthService.validateSmsCode` 将验证码存入 Redis，TTL 为 **300 秒**。
- 在测试过程中，当验证码过期后，通过 ADB 手动触发登录会收到 `DioException [bad response]: status code of 400`。
- **建议**: E2E 测试前需确保执行 `docker exec sw-redis redis-cli SET sms_code:13800138000 123456 EX 300`，或后端对 E2E 测试手机号延长/取消 TTL。

### 2. `profile/setup.yaml` 日期选择器本地化风险
- flow 中使用 `tapOn: "OK"` 确认日期选择器。若模拟器系统语言为中文，Flutter `showDatePicker` 的确认按钮文案为 `"确定"` 而非 `"OK"`，可能导致后续步骤失败。
- **建议**: 统一模拟器语言为英文，或在 flow 中使用 `point` / `id` 选择器避开文案差异。

### 3. `auth/login` 为假阳性通过
- 该 flow 没有任何登录成功后的断言，因此即使登录按钮未实际触发，Maestro 仍标记为通过。
- **建议**: 取消注释末尾的断言，并根据实际路由调整为 `assertVisible: "完善资料"`（新用户）或 `assertVisible: "首页"`（老用户）。
