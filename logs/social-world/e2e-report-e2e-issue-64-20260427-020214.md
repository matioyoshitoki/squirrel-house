## 失败根因分析（首次运行）

首次运行中 5 个 flow 失败，截图均显示应用停留在「完善资料 → 上传头像」页面，而非预期的首页或消息页。

| Flow | 失败原因 | 根因分类 | 修复方式 |
|------|---------|---------|---------|
| im/enter_chat | Element not found: `tab_messages` | Flow 过时/不完整 | 登录后补充条件性资料完善步骤 |
| im/send_message | Element not found: `tab_messages` | Flow 过时/不完整 | 登录后补充条件性资料完善步骤 |
| im/view_conversation_list | Element not found: `tab_messages` | Flow 过时/不完整 | 登录后补充条件性资料完善步骤 |
| matching/swipe_recommendations | Assertion false: `mode_tab_nearby` | Flow 过时/不完整 | 登录后补充条件性资料完善步骤 |
| matching/view_match_list | Element not found: `tab_messages` | Flow 过时/不完整 | 登录后补充条件性资料完善步骤 |

**根因**: 这些 flow 使用 `clearState: true` 启动，每次都会清除本地 token 并以全新会话登录。后端对测试手机号 `13800138000` 返回 `profileComplete=false`，导致登录后强制进入资料完善页。这些 flow 未包含资料完善步骤，因此后续对首页元素的断言/操作全部失败。

**修复策略**: 在 `includes/login-as-test-user.yaml` 之后、进入首页交互之前，统一插入条件分支：

```yaml
- runFlow:
    when:
      visible: "完善资料"
    file: includes/profile-setup-actions.yaml
```

该修复使所有 flow 具备自包含能力，无论后端用户状态如何均可独立运行。
