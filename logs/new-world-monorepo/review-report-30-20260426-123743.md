## 严重问题（Major）

### 1. 全部 UI 组件文件仍保留 `export default`，违反 F-11 强制规范
- **位置**: 全部 16 个 `apps/phaser3/src/ui/*.ts` 组件文件末尾
  - `UIPanel.ts:299` — `export default UIPanel;`
  - `UIDialog.ts:187` — `export default UIDialog;`
  - `UIToast.ts:407` — `export default UIToast;`
  - `UIAlert.ts:182` — `export default UIAlert;`
  - `UIButton.ts:163` / `UIIcon.ts:98` / `UIIconButton.ts:173` / `UIInput.ts:167` / `UIList.ts:164` / `UIListItem.ts:131` / `UIAvatar.ts:121` / `UIProgressBar.ts:123` / `UIResourceDisplay.ts:107` / `UIScrollView.ts:164` / `UISegmentedControl.ts:250` / `UIText.ts:46` / `UITooltip.ts:146`
- **问题**: 设计规范 `designs/issue-19/design-spec.md` 明确将 `export default` 列为 F-11 违反项（P-06、D-07、T-07、A-05），要求改为命名导出 `export class`。`docs/design-docs/ui-style-guide/03-component.md` 3.9 检查清单也强制要求：「组件文件使用命名导出 `export class`，不使用 `export default`」。`docs/modules/client/ui.md` 同样重申此规则。
- **影响**: 虽然 `index.ts` 使用命名导入，但文件内同时存在命名导出和默认导出会导致规范混乱，且给后续重构和 tree-shaking 带来隐患。
- **修复**: 从上述 16 个文件中删除每一行的 `export default Xxx;`。
- **依据**: `designs/issue-19/design-spec.md` 第 2 节问题分析表（P-06 / D-07 / T-07 / A-05）; `docs/design-docs/ui-style-guide/03-component.md` 第 289-296 行。

### 2. UIButton `setDisabled(false)` 无法恢复交互事件监听
- **位置**: `apps/phaser3/src/ui/UIButton.ts:84-141`（`setupInteraction`）及 `:149-160`（`setDisabled`）
- **问题**: 构造函数中若传入 `isDisabled: true`，`setupInteraction()` 会在 `:89-92` 提前 `return`，导致 `pointerover` / `pointerout` / `pointerdown` / `pointerup` 事件监听器**从未被注册**。后续调用 `setDisabled(false)` 仅执行 `target.setInteractive({ useHandCursor: true })` 和 `this.setAlpha(1)`，但事件监听仍然缺失，按钮虽可交互但无 Hover / Pressed / Click 响应。
- **影响**: 任何在构造时禁用、后续启用的 UIButton 将处于“假交互”状态，破坏状态机完整性。
- **修复**: 重构 `setupInteraction()`，使其**始终注册**事件监听，仅在回调内部判断 `this.config.isDisabled` 来决定是否执行视觉反馈和 `onClick`；`setDisabled()` 仅负责切换 `isDisabled` 标志和 `disableInteractive()` / `setAlpha()`。
  ```typescript
  // 示例修复思路
  target.on('pointerover', () => {
    if (this.config.isDisabled) return;
    killTweensOf(this.scene, this);
    ...
  });
  // 构造函数中不再因 isDisabled 提前 retur

  if (this.config.isDisabled) {
    target.disableInteractive();
    this.setAlpha(0.6);
  }
  ```
- **依据**: `docs/design-docs/ui-style-guide/03-component.md` 3.1 节要求「所有按钮组件都有 Normal / Hover / Pressed / Disabled 四态视觉区分」；PRD-004 3.6 组件状态机要求从 DISABLED 恢复后自动回到 NORMAL 态。

### 3. `apps/phaser3/src/ui/__tests__/` 缺少 `UIAlert.test.ts`
- **位置**: `apps/phaser3/src/ui/__tests__/`
- **问题**: PRD-004 成功指标要求「单元测试覆盖率 ≥ 80%（全部组件）」，且 `designs/issue-19/design-spec.md` 9.4 验收标准列明需测试 `show()` / `hide()` / `isAutoHide` / `getTypeColor()` 等。设计目录 `designs/issue-19/src/ui/__tests__/UIAlert.test.ts` 已存在较完善的测试（12 个 case），但生产代码目录 `apps/phaser3` 中未同步创建对应测试文件。
- **影响**: 生产测试覆盖不完整；后续 CI 无法验证 UIAlert 回归。
- **修复**: 将 `designs/issue-19/src/ui/__tests__/UIAlert.test.ts` 适配到 `apps/phaser3/src/ui/__tests__/UIAlert.test.ts`，使用 `MockScene` 替代 `UIMockScene`（或复用 `src/ui/__tests__/setup.ts` 中的 `UIMockScene`）。
- **依据**: PRD-004 第 1.2 节测试覆盖率指标；`designs/issue-19/design-spec.md` 9.4 验收标准。

### 4. UISegmentedControl 存在魔法数字色值且默认选中色与 PRD-004 不一致
- **位置**: `apps/phaser3/src/ui/UISegmentedControl.ts:138` 及 `:54`
- **问题**:
  1. **魔法数字**: 非选中标签颜色硬编码为 `'#aaaaaa'`（`:138`），违反 `docs/frontend/agent-design-guide.md` Step 3.2「颜色使用 `UITheme` 常量，禁止魔法数字」。
  2. **PRD 不一致**: 默认 `selectedColor` 为 `UITheme.stoneCyan`（`:54`），但 PRD-004 5.13 明确规定「选中态使用 `UITheme.indigoBlue`」。
- **影响**: 视觉风格与国风色板不一致；若主题色调整，该组件不会同步变更。
- **修复**:
  ```typescript
  // :54
  selectedColor: config.selectedColor ?? UITheme.indigoBlue,
  // :138
  color: colorToHex(UITheme.lightInk),  // 替代 '#aaaaaa'
  ```
- **依据**: `docs/frontend/agent-design-guide.md` 第 114 行；PRD-004 5.13 变更要点。

### 5. UIAlert `hide()` 动画时长 150ms，与 PRD-004 及设计规范要求的 200ms 不符
- **位置**: `apps/phaser3/src/ui/UIAlert.ts:148`
- **问题**: `UIAlert.hide()` 使用 `duration: UITweens.fade.duration`（150ms）。但 `prd/PRD-004-ui-component-library-redesign.md` 5.15 明确规定退场动画 `duration=200ms`；`designs/issue-19/design-spec.md` 3.4 节同样写明 `duration=200ms`。
- **影响**: UIAlert 退场动画比规范快 50ms，与其他弹窗/提示组件的视觉节奏不一致。
- **修复**: 将 `UIAlert.hide()` 中 `duration` 改为字面量 `200`（或新增 `UITweens.alertOut` 配置）。
  ```typescript
  duration: 200,  // 替代 UITweens.fade.duratio

  ```
- **依据**: `prd/PRD-004-ui-component-library-redesign.md` 5.15；`designs/issue-19/design-spec.md` 3.4 动画标准表。

## 风险与建议

1. **export default 遗留风险**: 虽然当前 `index.ts` 统一使用命名导出，但文件级 `export default` 的长期存在会诱使后续开发者使用默认导入，破坏规范一致性。建议一次性全部清理。

2. **UIButton 状态机脆弱性**: `setDisabled(false)` 的事件监听缺失是**状态机实现缺陷**，在动态权限/表单校验场景中极易触发。建议将事件监听注册与 `isDisabled` 状态判断解耦。

3. **设计资产与生产代码的测试漂移**: `designs/issue-19/src/ui/__tests__/` 中的测试比 `apps/phaser3/src/ui/__tests__/` 更完善（使用了 `UIMockScene`、`expectTweenCreated` 等 helper）。建议建立规范：设计目录的测试在合并前必须同步到生产目录，或统一从生产目录运行。

4. **文档三处不一致**: PRD-004（200ms）/ design-spec（200ms）/ ui-style-guide（150ms）对 UIAlert 退场时长的描述不一致，应在修复代码后统一文档。

5. **建议补充的测试场景**（非阻塞，但建议后续迭代）：
   - UIPanel: `show()` 在 `hide()` 动画中途被调用的行为（killTweensOf + 新动画）。
   - UIDialog: 确认回调确实在 `hide` 动画 `onComplete` 后触发（当前 `apps/phaser3` 的测试仅检查 `dialog` 被定义，未触发按钮）。
   - UIToast: ToastManager 队列满时最早 Toast 被 dismiss 后，剩余 Toast 的 reposition 验证。
   - UIAlert: `getTypeColor` 静态方法的四态返回值验证（设计目录已有，生产目录缺失）。
