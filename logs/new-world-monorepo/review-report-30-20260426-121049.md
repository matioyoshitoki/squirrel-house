## 审查结论

审查结论: **NEEDS_MAJOR_FIX**

PR #30（branch: `design/issue-19`）包含两部分内容：
1. `designs/issue-19/` 下新的设计交付物（UIPanel / UIDialog / UIToast / UIAlert / UIButton 及基础设施）
2. `apps/phaser3/src/ui/` 下现有组件的集成更新（UIBadge、UIButton、UISegmentedControl、UIAlert、UIPanel、UIDialog、UIToast 等）

设计交付物整体质量较高，但生产代码（`apps/phaser3/src/ui/`）存在大量与 PRD-004 及设计规范不一致的遗留实现未清理问题，且部分核心架构要求（如 UIDialog 继承 UIPanel）未在生产代码中落地。

## 阻塞问题（Blocking）

### 1. UIDialog 生产代码未继承 UIPanel（核心架构违反）
- **位置**: `apps/phaser3/src/ui/UIDialog.ts:22`
- **问题**: `UIDialog` 仍 `extends Phaser.GameObjects.Container`，自行实现背景/标题/遮罩/按钮。设计规范 `designs/issue-19/design-spec.md` 2.2 节 D-01 明确要求「**必须继承 UIPanel**」，PRD-004 5.6 节同样要求继承以复用遮罩和动画。
- **影响**: 遮罩逻辑、show/hide 动画、深度管理在 UIDialog 和 UIPanel 中重复实现且参数不一致；后续维护需要改两处。
- **修复**: 将 `apps/phaser3/src/ui/UIDialog.ts` 重构为 `extends UIPanel`，复用 `show()`/`hide()`/遮罩/关闭逻辑，删除自行实现的 `createOverlay()`/`createBackground()`/`createTitle()`。
- **依据**: `designs/issue-19/design-spec.md` D-01 / PRD-004 5.6

### 2. UIPanel 关闭按钮仍使用 Unicode `×`（F-01 违反）
- **位置**: `apps/phaser3/src/ui/UIPanel.ts:86`
- **问题**: `createCloseButton()` 中使用 `this.scene.add.text(closeX, closeY, '×', ...)` 渲染关闭按钮。`designs/issue-19/design-spec.md` 2.1 节 P-02 及 `docs/design-docs/ui-style-guide/03-component.md` 3.3 均明确禁止使用 `UIText` 渲染 `×` 字符，要求使用 `UIIcon` 加载 `cancel-btn-0.png`。
- **影响**: 直接违反国风水墨风格规范 F-01（emoji/Unicode 图标禁令）。
- **修复**: 参照 `designs/issue-19/src/ui/UIPanel.ts:160-221` 的实现，将关闭按钮改为 `UIIcon` 或 `Phaser.GameObjects.Image` 加载 `cancel-btn-0.png`，Hover 时边框变为 `UITheme.cinnabarRed`。
- **依据**: `designs/issue-19/design-spec.md` P-02 / `docs/design-docs/ui-style-guide/03-component.md` 3.3

### 3. UIAlert 仍使用 Unicode 符号作为类型图标（F-01 违反）
- **位置**: `apps/phaser3/src/ui/UIAlert.ts:63-70`
- **问题**: `getTypeIcon()` 返回 `'✓'`、`'!'`、`'×'`、`'i'` 等 Unicode 符号，并在 `createIcon()` 中渲染。`designs/issue-19/design-spec.md` 2.4 节 A-01 及 `docs/design-docs/ui-style-guide/03-component.md` 3.7 均明确禁止此行为，要求「**删除图标文字，仅用左侧 4px 色条标识类型**」。
- **影响**: 违反 F-01，与设计稿不一致。
- **修复**: 删除 `getTypeIcon()` 和 `createIcon()` 方法，仅保留左侧 4px 色条（已在 `createBackground()` 中部分实现，但需删除多余的图标文字）。
- **依据**: `designs/issue-19/design-spec.md` A-01 / `docs/design-docs/ui-style-guide/03-component.md` 3.7

### 4. UIToast 生产代码未按 PRD-004 重做（类型体系/深度/动画/色条全错）
- **位置**: `apps/phaser3/src/ui/UIToast.ts`
- **问题**: 
  - 仍使用业务语义 `ToastType = 'build' | 'upgrade' | 'remove' | ...`，未迁移为 `UINotificationType`（设计规范 T-08）
  - `depth = 4000`（应为 5000+，设计规范 T-06）
  - 缺少左侧 4px 类型色条（设计规范 T-02）
  - `show()` 动画 duration 为 200ms（PRD-004 3.4 要求 300ms）
  - `hide()` 动画方向为 `y + 20`（向下，PRD-004 要求 `y 0 → -10` 向上），duration 300ms（应为 200ms）
  - `ToastManager` 非单例，无 `getInstance()`（设计规范 T-05）
  - 未使用 `killTweensOf` 防止动画叠加（PRD-004 3.4）
- **影响**: 生产代码的视觉与行为与新版规范完全脱节；`designs/issue-19` 中的正确实现未同步到 `apps/phaser3`。
- **修复**: 将 `apps/phaser3/src/ui/UIToast.ts` 替换为 `designs/issue-19/src/ui/UIToast.ts` 的实现（或等效重构），保留必要的旧接口兼容性包装。
- **依据**: `designs/issue-19/design-spec.md` T-01~T-08 / PRD-004 3.4、5.11

### 5. UIButton 生产代码缺少 pressedColor 与 killTweensOf（PRD-004 违反）
- **位置**: `apps/phaser3/src/ui/UIButton.ts:11-21`（接口）、`77-122`（交互）
- **问题**: 
  - `UIButtonConfig` 缺少 `pressedColor?: number`（PRD-004 5.3 明确要求）
  - `pointerover`/`pointerout`/`pointerdown` 均**未**在 tween 前调用 `killTweensOf`（PRD-004 3.4 禁止项：「禁止连续触发时动画叠加，必须先 killTweensOf」）
  - `pointerout` 恢复 Normal 时硬编码 `fillAlpha: 0.9`（`docs/design-docs/ui-style-guide/03-component.md` 3.1 要求「应读取 `this.background.fillAlpha` 恢复，禁止硬编码」）
- **影响**: 快速 hover/click 时动画叠加；Pressed 态无独立颜色支持。
- **修复**: 
  1. 在 `UIButtonConfig` 中增加 `pressedColor?: number`
  2. 所有 tween 播放前增加 `killTweensOf(this.scene, this)`
  3. `pointerout` 时记录并恢复原始 `fillAlpha`
- **依据**: PRD-004 5.3、3.4 / `docs/design-docs/ui-style-guide/03-component.md` 3.1

## 严重问题（Major）

### 6. UIPanel 生产代码缺少 show/hide 动画与模态遮罩
- **位置**: `apps/phaser3/src/ui/UIPanel.ts`
- **问题**: 生产代码的 `UIPanel` 仅有 `getContentBounds()`/`addContent()`，**缺少** `show()`/`hide()` 标准动画、`showOverlay`/`overlayAlpha`/`isModal`/`onOverlayClick` 等配置（设计规范 P-01、P-04、P-05）。由于 UIDialog 未继承 UIPanel，这些缺失被进一步放大。
- **修复**: 将生产代码 `UIPanel` 升级至与设计规范一致，或至少在现有代码中补全 `show()`/`hide()` 与遮罩能力。
- **依据**: `designs/issue-19/design-spec.md` P-01、P-04、P-05 / PRD-004 5.5

### 7. UIDialog 生产代码 confirm/cancel 回调在 hide 动画之前执行
- **位置**: `apps/phaser3/src/ui/UIDialog.ts:152-160`
- **问题**: 
  ```ts
  private confirm(): void {
    this.hide();
    this.config.onConfirm?.();  // ← hide 动画刚开始就执行回调
  }
  ```
PRD-004 5.6 及设计规范 3.2 均要求「先播放 hide 动画，动画**完成后**再执行 `onConfirm`」。
- **修复**: 修改 `confirm()`/`cancel()` 为 `this.hide(() => this.config.onConfirm?.())`，确保 `hide` tween 的 `onComplete` 中触发回调。
- **依据**: PRD-004 5.6 / `designs/issue-19/design-spec.md` 3.2

### 8. UIDialog 生产代码 depth 为 5000（应为 3000）
- **位置**: `apps/phaser3/src/ui/UIDialog.ts:53`
- **问题**: `this.setDepth(5000)`，但 PRD-004 3.5 及设计规范均要求 Dialog 层级为 3000。
- **修复**: 改为 `this.setDepth(UIDepthLayer.Dialog)` 或使用 `getNextDepth(UIDepthLayer.Dialog, scene)`。
- **依据**: PRD-004 3.5 / `designs/issue-19/design-spec.md` D-03

### 9. UIAlert 生产代码缺少 show() 方法与正确退场动画
- **位置**: `apps/phaser3/src/ui/UIAlert.ts`
- **问题**: 
  - 无 `show()` 方法（设计规范 A-02 要求增加显式 show 接口含入场动画）
  - `hide()` 中 `y: this.y - 20`（应为 `y: this.y - 10`，PRD-004 3.4 及设计规范 3.4）
  - `static show()` 未调用 `alert.show()`（只是 new 后返回）
  - 未使用 `getNextDepth`，depth 硬编码为 `UIDepthLayer.Toast`（虽满足最低要求，但无法处理多个 Alert 叠加）
- **修复**: 参照 `designs/issue-19/src/ui/UIAlert.ts` 补全 `show()` 与正确的 `hide()` 动画参数。
- **依据**: `designs/issue-19/design-spec.md` A-02、A-03 / PRD-004 3.4

### 10. UIBadge 修改后无配套单元测试
- **位置**: `apps/phaser3/src/ui/UIBadge.ts`

## 风险与建议

1. **生产代码与设计稿严重脱节的风险**：本 PR 的最大风险是 `apps/phaser3/src/ui/` 下的组件仅做了「兼容性补丁」式修改（如改 import、改命名、加 createFallbackRect），而 PRD-004 要求的结构性重做（UIDialog 继承 UIPanel、Toast 类型体系更换、Alert/Toast 左侧色条、show/hide 标准动画）**完全没有在生产代码落地**。这会导致后续业务模块（Fish/Weapon/Market 等）基于旧版不一致的接口开发，技术债务进一步扩大。

2. **建议合并策略**：
   - **方案 A（推荐）**：在合并前，要求将 `designs/issue-19/src/ui/` 中经过测试验证的实现（UIPanel、UIDialog、UIToast、UIAlert）**完整迁移**到 `apps/phaser3/src/ui/`，替换旧实现，并确保 `city-builder/ui/SceneSwitchBar.ts` 等调用方兼容。
   - **方案 B**：如果本 PR 的定位仅为「设计交付 + 文档更新」，则应明确将 `apps/phaser3/src/ui/` 中的修改拆出，另开 Integration PR，避免给代码库留下半重做的混乱状态。

3. **测试建议**：在将设计稿迁移到生产代码时，优先复用 `designs/issue-19/src/ui/__tests__/setup.ts` 中的 `UIMockScene` 和 helper，并统一替换 `apps/phaser3/src/ui/__tests__/setup.ts` 中的旧基础设施。
