## 严重问题（Major）

### 1. UIIcon `onClick` 在 `pointerdown` tween 的 `onComplete` 中触发，违反交互事件流规范
- **位置**: `apps/phaser3/src/ui/UIIcon.ts:73-85`
- **问题**: `UIIcon` 将 `onClick` 放在 `pointerdown` 动画的 `onComplete` 中调用，而非 `pointerup`。这导致：
  1. 交互延迟约 100ms（50ms press + 50ms yoyo），快速点击手感迟滞；
  2. 鼠标划出后（`pointerupoutside`）仍会触发点击；
  3. 与 `UIButton` 在 `pointerup` 立即触发 `onClick` 的行为不一致。
- **影响**: 使用 `UIIcon` 作为关闭按钮（如 `UIPanel` 的 `cancel-btn-0`）或其他交互入口时，玩家体验与预期不符。
- **修复**:
  ```typescript
  // apps/phaser3/src/ui/UIIcon.ts
  // 1. pointerdown 仅处理视觉反馈
  target.on('pointerdown', () => {
    killTweensOf(this.scene, this);
    this.scene.tweens.add({
      targets: this,
      scale: 0.95,
      duration: UITweens.press.duration,
      ease: UITweens.press.ease,
      yoyo: true,
    });
  });

  // 2. pointerup 恢复视觉并立即触发 onClick
  target.on('pointerup', () => {
    if (this.config.isInteractive) {
      this.config.onClick?.();
    }
  });

  // 3. pointerupoutside 取消 Pressed 态，不触发 onClick
  target.on('pointerupoutside', () => {
    killTweensOf(this.scene, this);
    this.setScale(1);
  });
  ```
- **依据**: 本轮 PR 在 `docs/design-docs/ui-style-guide/03-component.md` 中**新增**的条款：「禁止将 `onClick` 放到 `pointerdown` tween 的 `onComplete` 中触发，否则会产生 ~100ms 的交互延迟」；`docs/frontend/agent-design-guide.md` 常见陷阱表也明确禁止此模式。

### 2. UIToast `dismiss()` 与 UIAlert `hide()` 使用相对 y 坐标，存在累积漂移风险
- **位置**: 
  - `apps/phaser3/src/ui/UIToast.ts:258`
  - `apps/phaser3/src/ui/UIAlert.ts:154`
- **问题**: 退场动画目标写为 `y: this.y - 10`，依赖当前 `this.y` 的瞬时值。若在 `show()` 动画未完成时调用 `dismiss()` / `hide()`，`this.y` 处于 `baseY + 20` 到 `baseY` 之间的某个中间值，导致 tween 目标错误；重复调用时坐标会持续漂移。
- **影响**: 快速连续显示/隐藏 Toast 或 Alert 时，组件位置偏离设计锚点（底部居中 / 顶部居中）。
- **修复**:
  ```typescript
  // UIToast.ts:258
  y: this.baseY - 10,

  // UIAlert.ts:154
  y: this.baseY - 10,
  ```
- **依据**: 本轮 PR 在 `docs/design-docs/ui-style-guide/03-component.md` 中**新增**的条款：「`show()` 幂等性：...必须在构造时保存原始目标坐标（`this.baseY = this.y`），`show()` 的 tween 目标使用 `this.baseY` 而非 `this.y - 20`。禁止依赖 `this.y` 的当前值计算 tween 目标」；`docs/frontend/agent-design-guide.md` 也将「`show()` / `hide()` 动画基于当前状态做相对偏移」列为常见陷阱。

### 3. UIAlert `hide()` 动画时长与 PRD-004 不一致
- **位置**: `apps/phaser3/src/ui/UIAlert.ts:155`
- **问题**: 代码使用 `duration: UITweens.fade.duration`（值为 150ms），但 PRD-004 5.15 明确规定退场动画 `duration=200ms`。
- **影响**: 视觉验收时会被判定为不符合规范。
- **修复**:
  ```typescript
  // apps/phaser3/src/ui/UIAlert.ts:155
  duration: 200, // 或新增 UITweens.alertOut = { duration: 200, ease: 'Power2' }
  ```
- **依据**: `prd/PRD-004-ui-component-library-redesign.md` 第 833 行（本轮 PR 增补）：「退场：`alpha: 1 → 0`，`y: 0 → -10`，`duration=200ms`，`ease='Power2'`」。

## 轻微问题（Minor）

### 4. UIIconButton 未实现 `textures` 背景纹理切换
- **位置**: `apps/phaser3/src/ui/UIIconButton.ts:81-88`
- **问题**: PRD-004 5.4 要求 `textures: { normal, hover, pressed, disabled }` 支持三态纹理切换，且 Hover 态缺失时使用 `scale=1.05` 动画替代。当前实现完全使用 Rectangle，未读取 `config.textures`。
- **修复**: 在 `createBackground()` 中增加纹理分支：若 `textures.normal` 存在且已加载，使用 `Phaser.GameObjects.Image`；否则使用 Rectangle 回退。

### 5. 生产代码中仍存在魔法数字色值
- **位置**:
  - `apps/phaser3/src/ui/UIResourceDisplay.ts:41` — `bgColor: config.bgColor ?? 0xffffff`（应改为 `UITheme.white`）
  - `apps/phaser3/src/ui/UIScrollView.ts:71` — `0x333333`（应改为 `UITheme.darkInk`）
  - `apps/phaser3/src/city-builder/ui/TopInfoBar.ts:102-103` — `bgColor: 0x333333`, `strokeColor: 0x666666`（应改为 `UITheme.darkInk` / `UITheme.lightInk`）
- **说明**: `UIResourceDisplay.ts` 和 `UIScrollView.ts` 的魔法数字在旧版已存在，但本轮 PR 对这两个文件做了修改（引入 `UITheme` 并部分替换色值），理应一并清理。`TopInfoBar.ts` 的 UIAvatar 配置未随本轮迁移同步替换为 `UITheme` 常量。

### 6. designs/issue-19 与 apps/phaser3/src/ui 存在实现漂移
- **问题**: `designs/issue-19/src/ui/` 中的文件与生产代码存在差异（如 `UIPanel.closeButton` 实现方式、`UIToast` 旧 API 兼容逻辑等）。虽然 `designs/issue-19/design-spec.md` 已声明「实现以 `apps/phaser3/src/ui/` 为准」，但 `designs/` 目录的错误代码仍可能成为后续 Agent 的参考源。
- **修复**: 反向同步 `apps/phaser3/src/ui/` 的最终实现到 `designs/issue-19/src/ui/`，或在 `design-spec.md` 中逐条列出已知差异。

### 7. MockTweenManager `onComplete` 同步执行，测试存在假阳性风险
- **位置**: `apps/phaser3/src/test-setup.ts:349-351`
- **问题**: `MockTweenManager.add()` 在创建 tween 后立即同步调用 `config.onComplete()`，导致所有依赖「动画完成后回调」的测试无法验证真实时序。本轮 PR 已在 `docs/frontend/agent-design-guide.md` 中记录此陷阱，但未修复 `test-setup.ts`。
- **说明**: 这不是本轮 PR 引入的新问题，但建议在本次大范围 UI 重构中一并修复，或至少为 tween mock 增加一次 `setImmediate` / 事件循环延迟。

### 8. city-builder/ui/ 文件仍保留 `export default`
- **位置**: `apps/phaser3/src/city-builder/ui/*`（13 个文件）
- **问题**: `src/ui/` 组件已全部改为命名导出，但 `city-builder/ui/` 业务组件仍有 `export default`。虽不属于 Layer 1 原子组件的强制范围，但风格不一致。
- **说明**: 如不在本轮修复，建议在技术债务追踪器中记录。

## 风险与建议

1. **动画漂移风险（高）**: UIToast 与 UIAlert 的相对 y 偏移在真实场景中（如网络抖动导致消息重复显示、玩家快速操作）可能产生肉眼可见的位置漂移。建议在修复后补充「连续 show/hide 3 次」的单元测试，断言最终 `y === baseY`。

2. **UIIcon 交互延迟风险（中）**: 由于 `UIPanel` 的关闭按钮使用 `UIIcon` 实现，当前约 100ms 的点击延迟会影响所有面板的关闭体验。修复后应验证 `pointerup` 触发逻辑与 `UIButton` 一致。

3. **设计资产漂移风险（中）**: `designs/issue-19/` 与生产代码的差异可能导致后续 Agent 复制错误实现。建议在合并前反向同步或显式标注差异清单。

4. **测试执行风险（中）**: 审查环境无法运行 `pnpm test` 和 `tsc --noEmit`，无法确认测试全绿和类型检查通过。建议提交者在 PR 描述中补充 CI 通过截图或测试报告。

5. **UIIconButton 纹理缺失风险（低）**: 当前实现完全回退到 Rectangle，若美术资源已就绪但代码未读取 `textures`，上线后会出现「所有图标按钮均为纯色方块」的视觉效果。建议在业务 Scene 视觉验收时重点检查。
