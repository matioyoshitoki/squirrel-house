## 严重问题（Major）

### 1. UIDialog 消息文字颜色与 Design Spec 不一致
- **位置**: `apps/phaser3/src/ui/UIDialog.ts:113`
- **问题**: 消息正文使用 `colorToHex(UITheme.lightInk)`（`#666666`），但 `designs/issue-19/design-spec.md` 第 3.2 节明确规定消息文字颜色应为 `#cccccc`（即 `0xcccccc`）。新颜色比规范暗了约 60%，在 `UITheme.inkBlack` 背景上对比度虽足够，但视觉上明显偏离设计稿。
- **影响**: 所有使用 UIDialog 的提示信息可读性下降，视觉验收不通过。
- **修复**: 
  ```typescript
  // apps/phaser3/src/ui/UIDialog.ts:113
  getInkTextStyle('14px', colorToHex(0xcccccc), {
  ```
同步修改 `designs/issue-19/src/ui/UIDialog.ts:111` 保持一致。
- **依据**: `designs/issue-19/design-spec.md` 第 168 行："消息文字: 14px，颜色 #cccccc（即 0xcccccc），居中，自动换行"

### 2. UIPanel 遮罩层深度实现违反 Depth 规范
- **位置**: `apps/phaser3/src/ui/UIPanel.ts:97-116`
- **问题**: 遮罩层（overlay）通过 `this.add(this.overlay)` 被加入 Panel 容器内部，随后调用 `this.overlay.setDepth(UIDepthLayer.Overlay)`（4000）。但在 Phaser 3 中，**容器子对象的 `depth` 属性不参与全局深度排序**——子对象始终随容器一起渲染，全局有效深度等于容器自身的深度。这意味着：
  1. 遮罩层的 `setDepth(4000)` 实际无效，它随 Panel 一起被渲染在 2000 或 3000 深度；
  2. 若存在非模态 Panel 被分配到 3001 深度，该 Panel 会渲染在 Dialog（3000）及其遮罩之上，破坏模态隔离。
- **影响**: 多面板/多弹窗并发场景下，模态遮罩无法保证最上层阻断交互，可能导致玩家透过遮罩操作底层界面。
- **修复**: 遮罩层应直接加入 Scene（而非 Panel 容器），并通过独立引用管理其生命周期，确保其深度真正为 4000 且不受 Panel 缩放/透明动画影响：
  ```typescript
  // 在 UIPanel 中不执行 this.add(this.overlay)
  // 改为由 scene 直接管理，或在 show()/hide() 中同步其显隐
  this.scene.add.existing(this.overlay);
  this.overlay.setDepth(UIDepthLayer.Overlay);
  ```
需在 `destroy()` 和 `hide()` 中显式销毁/隐藏 overlay。
- **依据**: `docs/design-docs/ui-style-guide/01-foundation.md` 第 120 行："同一深度层级的多个组件同时显示时，后创建的组件应自动递增 depth（+1）"；`designs/issue-19/design-spec.md` 第 3.1 节："遮罩层：始终 UIDepthLayer.Overlay（4000）"

### 3. designs/issue-19 测试断言错误（warning 色值）
- **位置**: `designs/issue-19/src/ui/__tests__/UIAlert.test.ts:77`
- **问题**: 测试断言 `expect(UIAlert.getTypeColor('warning')).toBe(UITheme.goldBrown)`，但 PRD-004 和 `docs/design-docs/ui-style-guide/03-component.md` 均规定 warning 类型色为 `UITheme.warning`（`0xffaa00`）。`apps/phaser3/src/ui/__tests__/UIAlert.test.ts:76` 已修正为 `UITheme.warning`，但设计交付物未同步。
- **影响**: 设计交付物与实现代码不一致，后续若按 designs 目录作为参考源会导致回归错误。
- **修复**: `designs/issue-19/src/ui/__tests__/UIAlert.test.ts:77` 改为：
  ```typescript
  expect(UIAlert.getTypeColor('warning')).toBe(UITheme.warning);
  ```
- **依据**: `docs/design-docs/ui-style-guide/03-component.md` 第 212 行：`warning | UITheme.warning`；`prd/PRD-004-ui-component-library-redesign.md` 第 5.15 节

## 风险与建议

1. **遮罩层深度风险（推荐后续计划修复）**: 当前遮罩作为容器子对象的实现虽然视觉表现正常（因为遮罩尺寸为 `camera.width * 2`），但在多弹窗并发或拖拽场景下存在层级错乱隐患。建议在下一个迭代中将遮罩提取为 Scene 级对象，由 `UIPanel` / `UIDialog` 引用管理，而非作为容器子对象。

2. **city-builder 旧组件迁移风险**: `city-builder/ui/` 中仍有大量组件使用旧 API（如 `export default`、魔法数字色值）。PRD-004 的验收标准要求 "city-builder/ui/ 旧组件迁移到新版 src/ui/"，建议建立独立 Issue 跟踪。

3. **UIButton 交互事件可靠性**: 当前 `pointerup` 直接触发 `onClick`，未校验 `pointerdown` 是否发生在同一按钮上。标准做法是增加一个 `isPressed` 标志位，仅在 `pointerdown` 后置位、`pointerup`/`pointerupoutside` 后复位，确保 `onClick` 只在完整按压周期内触发。当前实现虽非回归，但建议优化。

4. **设计资产一致性**: 建议建立规则——`designs/issue-XX/` 目录在 PR 合并前必须与 `apps/phaser3/src/` 中对应文件 `diff` 为零（或差异仅含注释/文档），避免设计交付物腐烂。
