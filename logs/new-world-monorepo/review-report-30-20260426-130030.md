## 严重问题（Major）

### 1. UIIcon 的 pointerdown 动画违反「动画铁律」—— 未执行 `killTweensOf`
- **位置**: `apps/phaser3/src/ui/UIIcon.ts:73-83`
- **问题**: `UIIcon` 的 `pointerdown` 事件处理器直接创建 tween，未在播放前调用 `killTweensOf`。根据 `docs/design-docs/ui-style-guide/03-component.md` 3.2 节「动画铁律」：
  > **Pressed tween 播放前必须先 `killTweensOf(this.scene, this)`。**
快速连续点击时，tween 会叠加，导致缩放动画异常。
- **影响**: 所有使用 `UIIcon` 且开启 `isInteractive` 的交互图标（如关闭按钮、道具图标）在连续点击时动画冲突。
- **修复**: 在 `pointerdown` 回调第一行添加 `killTweensOf(this.scene, this);`：
  ```typescript
  target.on('pointerdown', () => {
    killTweensOf(this.scene, this); // 新增
    this.scene.tweens.add({
      targets: this,
      scale: 0.95,
      duration: UITweens.press.duration, // 建议同步改为 UITweens.press
      ease: UITweens.press.ease,
      yoyo: true,
      onComplete: () => {
        this.config.onClick?.();
      },
    });
  });
  ```
- **依据**: `docs/design-docs/ui-style-guide/03-component.md` 第 76-78 行；PRD-004 3.4 节「禁止连续触发时动画叠加，必须先 `killTweensOf(target)`」。

## 轻微问题（Minor）

### 2. UIListItem 图标缺失时未执行资源回退契约
- **位置**: `apps/phaser3/src/ui/UIListItem.ts:71-78`
- **问题**: 当传入 `icon` 但纹理不存在时，代码仅跳过创建图标：
  ```typescript
  if (icon) {
    if (hasTexture(this.scene, icon)) {
      // 创建 image
    }
    // 无 else 分支：既没有 fallback Rectangle，也没有 console.war

    textX = 10;
  }
  ```
这违反了 PRD-004 3.3 节资源回退统一标准：
  > 1. **检查**：`if (texture && hasTexture(scene, texture))`
  > 2. **回退**：使用 `createInkBackground()` 或 `scene.add.rectangle()` + `setStrokeStyle()`
  > 3. **告警**：`console.warn(\`[UI] fallback: ${textureKey}\`)`
- **影响**: 列表项在缺少图标纹理时左侧出现空白偏移，且无回退提示，不利于资源缺失排查。
- **修复**: 补充 `else` 分支：
  ```typescript
  if (icon) {
    if (hasTexture(this.scene, icon)) {
      const iconImage = this.scene.add.image(-width / 2 + 20, 0, icon);
      iconImage.setDisplaySize(18, 18);
      iconImage.setOrigin(0.5);
      elements.push(iconImage);
    } else {
      const fallback = createFallbackRect(this.scene, 18, 18, icon, UITheme.lightInk, UITheme.goldBrown);
      fallback.setPosition(-width / 2 + 20, 0);
      elements.push(fallback);
    }
    textX = 10;
  }
  ```
同时需要在文件顶部引入 `createFallbackRect`。
- **依据**: PRD-004 3.3 节「资源回退统一标准」；`apps/phaser3/src/ui/utils.ts` 中 `createFallbackRect` 的实现。

### 4. ToastManager 构造器应为 private（设计规范偏差）
- **位置**: `apps/phaser3/src/ui/UIToast.ts:317`
- **问题**: 设计规范 `designs/issue-19/design-spec.md` 及 PRD-004 均要求 `ToastManager` 为单例模式，构造器应为 `private constructor`。实际实现为 `constructor(scene: Scene, startY?: number)`（public），允许外部直接 `new ToastManager(scene)`，弱化了单例约束。
- **修复**: 将构造器改为 `private`：
  ```typescript
  private constructor(scene: Scene, startY?: number) { ... }
  ```
现有测试中的 `new ToastManager(scene)` 调用应替换为 `ToastManager.getInstance(scene)`。
- **依据**: `designs/issue-19/design-spec.md` 第 221-224 行；PRD-004 5.11 ToastManager 接口设计。

## 风险与建议

1. **向后兼容风险**：`UISegmentedControl` 的 API 发生了破坏性变更（`selectedId` → `selectedIndex`/`key`，`onSelect` → `onChange`）。`SceneSwitchBar.ts` 已同步迁移，但建议全局搜索确认无其他业务代码使用旧 API。
2. **UIToast 旧 API 兼容层**：当前保留了 `title`、`startTime`、`endTime`、`progress`、`showProgress`、`onComplete`、`onClick` 等废弃字段。建议在后续迭代中标记具体移除版本，避免兼容层长期累积。
3. **深度计数器未重置**：`getNextDepth` 使用 Scene 级 WeakMap 计数器，但仅在 `ToastManager.clear()` 等极少数场景重置。如果同一 Scene 反复创建/销毁大量同层级组件，depth 可能无限增长（虽然实际场景中 5000+ 的 depth 在 Phaser3 中通常不会导致问题）。
4. **缺少 Layer 2 组件**：PRD-004 规划了 6 个 Layer 2 组件（UISlider、UIToggle、UIStepper、UIDropdown、UIPagination、UILoading），本次 PR 未交付。建议在后续 PR 中继续推进，并在 `docs/modules/client/ui.md` 的「后续计划」中保持更新。
