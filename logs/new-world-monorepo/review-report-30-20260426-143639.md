## 严重问题（Major）

### 1. UIAlert.hide() 动画时长与规范不一致（150ms vs 200ms）
- **位置**: `apps/phaser3/src/ui/UIAlert.ts:150`
- **问题**: `UIAlert.hide()` 使用了 `UITweens.fade.duration`（150ms）作为退场动画时长，但 `designs/issue-19/design-spec.md` 第 3.4 节明确要求退场动画 `duration=200ms`；PRD-004 第 5.15 节同样规定 `"退场：alpha 1 → 0，y - 10，duration=200ms，ease='Power2'"`。
- **影响**: UIAlert 退场动画比规范快了 50ms，在视觉验收环节会被判定为不符合动画标准。
- **修复**: 在 `UIAlert.hide()` 中将 `duration: UITweens.fade.duration` 改为 `duration: 200`（或复用 `UITweens.toastOut.duration`，因为 Toast 退场同样规范为 200ms）。
- **依据**: `designs/issue-19/design-spec.md` 第 3.4 节动画标准表格；`prd/PRD-004-ui-component-library-redesign.md` 第 5.15 节

### 2. UIDialog 消息文本使用魔法数字色值 0xcccccc
- **位置**: `apps/phaser3/src/ui/UIDialog.ts:109`
- **问题**: 消息正文颜色写死为 `colorToHex(0xcccccc)`，该色值未在 `UITheme` 中定义。`designs/issue-19/design-spec.md` 第 9.5 条验收标准明确要求 `"无魔法数字色值（grep -rn \"0x[0-9a-fA-F]\\{6\\}\" 仅命中 utils.ts）"`；`docs/frontend/agent-design-guide.md` Step 3.2 也强制要求 `"颜色使用 UITheme 常量，禁止魔法数字"`。
- **影响**: 违反项目编码规范，导致后续统一换肤或色板调整时无法自动同步。
- **修复**: 在 `apps/phaser3/src/ui/utils.ts` 的 `UITheme` 中新增一个语义化常量（如 `dialogMessage: 0xcccccc` 或 `mutedText: 0xcccccc`），然后在 `UIDialog.ts` 中引用 `colorToHex(UITheme.mutedText)`。
- **依据**: `designs/issue-19/design-spec.md` 第 9.5 节；`docs/frontend/agent-design-guide.md` Step 3.2

### 3. ToastManager 队列上限与 reposition 行为缺少测试覆盖
- **位置**: `apps/phaser3/src/ui/__tests__/UIToast.test.ts`
- **问题**: `ToastManager` 实现了单例获取、队列满时驱逐最早 Toast（上限 3 个）、Toast 消失后剩余 Toast 自动补位（`repositionToasts`）等核心逻辑，但现有测试仅覆盖了基础 show/clear/register/unregister/singleton，未覆盖：
  1. 当已存在 3 个 Toast 时再 `show()`，最早的 Toast 会被 `dismiss()`；
  2. `repositionToasts()` 被正确调用且补位 tween 参数正确（duration=200ms, ease='Power2'）。
- **影响**: 队列管理和自动补位是 PRD-004 5.11 和设计规范 9.3 明确要求的验收项，缺失测试意味着回归风险。
- **修复**: 在 `UIToast.test.ts` 的 `describe('ToastManager')` 中追加：
  - `it('should dismiss earliest toast when queue reaches 3', ...)`
  - `it('should reposition remaining toasts after dismiss', ...)`
- **依据**: `prd/PRD-004-ui-component-library-redesign.md` 第 7.3 节测试要求；`designs/issue-19/design-spec.md` 第 9.3 节

## 轻微问题（Minor）

### 1. UIPanel 默认回调使用空函数而非 undefined
- **位置**: `apps/phaser3/src/ui/UIPanel.ts:74-75`
- **问题**: `onClose: config.onClose ?? (() => {})` 和 `onOverlayClick: config.onOverlayClick ?? (() => {})` 默认分配了无操作函数，导致每次创建面板都产生两个额外的函数对象。更简洁的做法是保留 `undefined`，在调用处使用可选链 `this.config.onClose?.()`（代码中已使用了可选链，因此空函数是多余的）。
- **影响**: 极微小的内存/性能开销，主要属于代码风格问题。
- **修复**: 将默认值改为 `undefined`，例如 `onClose: config.onClose ?? undefined`。
- **依据**: 代码中第 113 行和第 213 行已使用 `this.config.onOverlayClick?.()` 和 `this.config.onClose?.()`，说明可选链调用模式已建立。

### 2. UIToast 旧 API 兼容的 auto-show 逻辑可能意外触发
- **位置**: `apps/phaser3/src/ui/UIToast.ts:113-115`
- **问题**: 构造函数中 `if (config.title || config.startTime) { this.show(); }` 会在传入 `title` 时自动播放入场动画。对于同时使用新 API `message` 和旧字段 `title` 的调用方，Toast 会在 `new` 时立即显示，而不是由调用方决定何时 `show()`。
- **影响**: 边缘场景下的行为不符合预期，但已在代码注释中标记为 backward compatibility。
- **修复**: 将条件收紧为仅在纯旧 API 场景下触发，例如 `if (!config.message && (config.title || config.startTime)) { this.show(); }`。
- **依据**: PRD-004 第 5.11 节规定新 API 优先使用 `message`，`title` 已标记为 deprecated。

### 3. designs/issue-19 与 apps/phaser3/src/ui 存在轻微代码漂移
- **位置**: `designs/issue-19/src/ui/UIAlert.ts` 与 `apps/phaser3/src/ui/UIAlert.ts`
- **问题**: designs 目录下的 `UIAlert.ts` 缺少 `show()` 方法中 `autoHideEvent` 的清理逻辑（apps 版本已补充），且 `createBackground()` 中保留了未使用的 `const color = this.getTypeColor();`。根据设计规范，`apps/phaser3/src/ui/` 是最终代码源，designs 目录是设计交付物，但两者应保持同步以避免维护混乱。
- **影响**: 设计交付物与最终实现不一致，可能导致后续开发者参考 designs 目录时产生困惑。
- **修复**: 将 `designs/issue-19/src/ui/UIAlert.ts` 同步为与 `apps/phaser3/src/ui/UIAlert.ts` 一致。
- **依据**: `designs/issue-19/design-spec.md` 第 1 节明确指出 `"实现以 apps/phaser3/src/ui/ 为准"`。

## 风险与建议

1. **动画参数漂移风险**: 本次审查发现 UIAlert.hide() 使用了 150ms 而非规范要求的 200ms。建议对所有涉及动画时长的代码做一次集中扫描（grep `duration:` 在 `src/ui/` 下），确保没有类似的规范偏差遗留在 UIButton、UIIconButton 等组件中。

2. **魔法数字色值回潮风险**: `grep -rn "0x[0-9a-fA-F]\{6\}" apps/phaser3/src/ui/` 目前除了 `utils.ts` 和 `UIDialog.ts` 外，还命中了若干仅被删除 `export default` 的文件（如 `UIScrollView.ts`、`UIResourceDisplay.ts`）。这些文件在本次 PR 中未做实质性重写，色值问题属于历史债务。建议在下一次专门的重做 Issue 中统一清理，避免在本次 PR 中扩大范围。

3. **MockScene 的 tween onComplete 同步执行问题**: 现有测试中 `MockTweenManager.add()` 立即调用 `config.onComplete()`，导致 `UIDialog` 的 "先 hide 动画、后执行 onConfirm" 逻辑无法在测试中被真正验证。建议在测试基础设施 `apps/phaser3/src/test-setup.ts` 中支持异步 tween 完成（至少提供一个可手动推进的 tick 机制），否则动画时序类缺陷只能依赖人工代码审查发现。

4. **designs/issue-19 目录的维护策略**: 该目录包含与 `apps/phaser3/src/ui/` 近乎完全一致的组件副本。建议在下一次相关 PR 中删除 designs 目录中的 `.ts` 实现文件，仅保留 `design-spec.md` 和截图，以减少双轨维护成本。如确需保留，应在 CI 中增加 `diff` 校验，防止 designs 与源码持续漂移。
