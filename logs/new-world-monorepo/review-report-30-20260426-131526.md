## 严重问题（Major）

### 1. UIButton 缺少 PRD-004 规定的 `setTexture()` 和 `setColors()` 公开方法

- **位置**: `apps/phaser3/src/ui/UIButton.ts`（整文件，缺少方法实现）
- **问题**: PRD-004 5.3 节明确定义了 `UIButton` 的公开 API：
  ```typescript
  class UIButton {
    setText(text: string): void;
    setDisabled(isDisabled: boolean): void;
    setTexture(texture: string): void;          // ❌ 未实现
    setColors(bgColor: number, hoverColor: number, strokeColor: number): void;  // ❌ 未实现
  }
  ```
同时 `docs/modules/client/ui.md` 也已更新并 documenting 了这两个方法，但代码中不存在。
- **影响**: 业务代码按新版文档调用 `setTexture()` / `setColors()` 时将报运行时错误；PRD-004 接口一致性验收不通过。
- **修复**: 在 `UIButton` 类中补充：
  ```typescript
  setTexture(texture: string): void {
    this.config.texture = texture;
    // 销毁旧 background，按 createBackground() 逻辑重建
  }
  setColors(bgColor: number, hoverColor: number, strokeColor: number): void {
    this.config.bgColor = bgColor;
    this.config.hoverColor = hoverColor;
    this.config.strokeColor = strokeColor;
    if (this.background instanceof Phaser.GameObjects.Rectangle) {
      this.background.setFillStyle(bgColor, this.originalFillAlpha);
      this.background.setStrokeStyle(2, strokeColor);
    }
  }
  ```
- **依据**: PRD-004 5.3 节接口设计；`docs/modules/client/ui.md` 第 86–89 行。

### 2. UIAlert / UIToast 的 `warning` 类型色错误使用了 `UITheme.goldBrown`

- **位置**:
  - `apps/phaser3/src/ui/UIAlert.ts:70` / `UIAlert.ts:167`
  - `apps/phaser3/src/ui/UIToast.ts:129`
- **问题**: 当 `type === 'warning'` 时，代码返回 `UITheme.goldBrown` (`0xc9a227`)。但 PRD-004 5.15 节强制规定 warning 色为 `0xffaa00`，且 `UITheme.warning` 已定义为此值。文档 `docs/design-docs/ui-style-guide/03-component.md` 3.7/3.8 节虽然写了 `UITheme.goldBrown`，但该文档是在 PR 中新增/更新的，与 PRD-004 源规范冲突。应以 PRD-004 为准。
- **影响**: 警告提示条视觉色值与产品规范不一致；测试 `UIAlert.test.ts` 第 76–78 行也在验证错误的 `goldBrown`，导致测试成为“假阳性”。
- **修复**:
  1. 将 `UIAlert.getTypeColor()` 和 `UIToast.getTypeColor()` 中 `case 'warning'` 的返回值改为 `UITheme.warning`
  2. 同步修正 `docs/design-docs/ui-style-guide/03-component.md` 中 warning 色条说明
  3. 同步修正 `apps/phaser3/src/ui/__tests__/UIAlert.test.ts` 中断言：`expect(UIAlert.getTypeColor('warning')).toBe(UITheme.warning)`
- **依据**: PRD-004 5.15 节类型色标准表格：`warning: 0xffaa00`；`apps/phaser3/src/ui/utils.ts:30` 已定义 `warning: 0xffaa00`。

### 5. `UIDialog` 未将 `onClose` 回调传递给 `UIPanel` 基类

- **位置**: `apps/phaser3/src/ui/UIDialog.ts:47–69`
- **问题**: `UIDialogConfig` 通过 `Omit<UIPanelConfig, 'title' | 'showClose'>` 继承了 `onClose`，但 `UIDialog` 构造 `panelConfig` 时未传递 `onClose: config.onClose`，导致用户传入的 `onClose` 被静默丢弃。虽然 `UIDialog` 当前不显示关闭按钮（`showClose: false`），但 API 层面存在“无效参数”。
- **影响**: 调用方传入 `onClose` 时期望在面板关闭时触发，实际永远不会执行，造成难以调试的交互缺陷。
- **修复**: 在 `panelConfig` 中增加 `onClose: config.onClose`，或在 `UIDialogConfig` 中显式 `Omit<UIPanelConfig, 'title' | 'showClose' | 'onClose'>` 以移除该参数，避免误导。
- **依据**: PRD-004 5.6 节：`UIDialogConfig` 必须继承 `UIPanelConfig`（通过 `Omit` 排除），确保 Dialog 可复用 UIPanel 的全部配置能力。

## 轻微问题（Minor）

### 1. `ProgressToast.ts` 仍保留 `export default`

- **位置**: `apps/phaser3/src/city-builder/ui/ProgressToast.ts:140`
- **问题**: 文件末尾有 `export default ProgressToast;`，违反 F-11（组件文件必须使用命名导出 `export class`）。
- **修复**: 删除 `export default ProgressToast;`，统一使用命名导出。所有引用方应改为 `import { ProgressToast } from '...'`。
- **依据**: `docs/design-docs/ui-style-guide/03-component.md` 3.9 检查清单：“组件文件使用命名导出 `export class`，不使用 `export default`”。

### 2. `UIAlert.show()` 未清除已存在的 `autoHideEvent`

- **位置**: `apps/phaser3/src/ui/UIAlert.ts:119–133`
- **问题**: 若 `show()` 被多次调用，前一次创建的 `delayedCall` 不会被取消，导致旧的自动隐藏定时器仍在运行，可能引发重复 `hide()` 动画或过早销毁。
- **修复**: 在 `show()` 开头增加：
  ```typescript
  if (this.autoHideEvent) {
    this.autoHideEvent.remove();
    this.autoHideEvent = undefined;
  }
  ```
- **依据**: 与 `UIToast.show()` 的现有防御性处理保持一致（`UIToast.ts:220–224`）。

### 3. `UIButton` 默认文字色使用硬编码 `'#ffffff'`

- **位置**: `apps/phaser3/src/ui/UIButton.ts:44`
- **问题**: `textColor: config.textColor ?? '#ffffff'` 直接硬编码了 HEX 字符串，而规范要求通过 `UITheme` + `colorToHex()` 注入。
- **修复**: 改为 `textColor: config.textColor ?? colorToHex(UITheme.white)`。
- **依据**: `docs/design-docs/ui-style-guide/03-component.md` 3.1 禁止事项：“文字颜色通过 `getInkTextStyle()` + `colorToHex()` 注入，禁止硬编码 HEX 字符串”。

### 4. `UIToast` 构造器隐式自动播放 `show()`，与 designs 版本行为不一致

- **位置**: `apps/phaser3/src/ui/UIToast.ts:111`
- **问题**: 为了兼容旧 API，构造器内自动调用 `this.show()`。而 designs 版本和 PRD-004 的契约是：构造后显式调用 `show()` 或 `static show()` 才播放入场动画。这导致 `UIToast.static show()` 在 apps 版本中无法显式再调用 `toast.show()`（否则会重置动画）。
- **修复建议（可选）**: 在 `ToastManager.show()` 中显式调用 `toast.show()`，并将构造器内的自动 `show()` 逻辑改为仅在检测到旧 API 参数（如 `title`、`startTime`）时才触发，以减少新 API 的副作用。
- **依据**: `designs/issue-19/src/ui/UIToast.ts` 中 `static show()` 显式调用 `toast.show()`；PRD-004 5.11 接口设计未提及构造器自动播放。

### 5. `apps/phaser3/src/ui/__tests__/setup.ts` 中 `assertFallbackWarn` 未在 app 测试中使用

- **位置**: `apps/phaser3/src/ui/__tests__/setup.ts:117–130`
- **问题**: 虽然导出了 `assertFallbackWarn`，但当前所有 app 层测试文件（`UIAlert.test.ts`、`UIBadge.test.ts`、`UISegmentedControl.test.ts`、`UIToast.test.ts`）均未调用该辅助函数，导致纹理缺失回退的 `console.warn` 验证缺失。PRD-004 3.3 节要求“回退时打印 `[UI] fallback: ${textureKey}` warn”。
- **修复**: 在至少一个组件测试中补充 fallback 验证用例，例如：
  ```typescript
  const warnSpy = createFallbackWarnSpy();
  new UIButton(scene, { text: '测试', texture: 'non-existent' });
  assertFallbackWarn(warnSpy, 'non-existent');
  ```
- **依据**: PRD-004 3.3 资源回退统一标准第 3 条：“告警：`console.warn(\`[UI] fallback: ${textureKey}\`)`”。

## 风险与建议

1. **色值规范冲突风险**：`docs/design-docs/ui-style-guide/03-component.md` 在 PR 中被写入了 `warning: UITheme.goldBrown`，与 PRD-004 的 `0xffaa00` 冲突。建议在合并前统一以 PRD-004 为准，否则后续业务模块 Agent 会引用错误色值。
2. **设计产物编译风险**：`designs/issue-19/src/ui/index.ts` 当前无法编译。若 Developer Agent 直接复制该目录文件到 `apps/phaser3/src/ui/`，构建将失败。建议修复导出后重新验证 `tsc --noEmit`。
3. **UIToast 构造器副作用风险**：apps 版 `UIToast` 构造器自动 `show()` 是为了兼容旧版 `ProgressToast`，但长期维护中应通过显式调用消除隐式副作用。建议在后续清理旧 API 时一并移除构造器内的 `this.show()`。
4. **UIButton API 缺口风险**：文档与 PRD 已承诺 `setTexture()` / `setColors()`，但代码缺失。建议优先补齐，避免业务模块前端开发时遭遇 `TypeError: btn.setTexture is not a function`。
