## 严重问题（Major）

### 1. UIPanel 纹理缺失时未执行统一 fallback war

- **位置**: `apps/phaser3/src/ui/UIPanel.ts:97-103`
- **问题**: 当 `config.texture` 传入但纹理不存在时，组件直接回退到 `createInkBackground()`，未调用 `createFallbackRect()`，因此未打印 `[UI] fallback: ${textureKey}` warn。
- **影响**: 资源缺失时开发者无法通过控制台 warn 定位问题，违反 PRD-004 3.3 资源回退统一标准。
- **修复**: 将 `createBackground()` 中的 `else` 分支改为区分 "texture 为空" 和 "texture 非空但缺失"：
  ```typescript
  if (texture && hasTexture(this.scene, texture)) {
    // 使用纹理
  } else if (texture) {
    this.background = createFallbackRect(this.scene, width, height, texture, bgColor, strokeColor);
  } else {
    this.background = createInkBackground(this.scene, width, height, bgColor, strokeColor);
  }
  ```
- **依据**: PRD-004 3.3 节 "资源回退统一标准" 第 3 条：回退时必须打印 `console.warn(\`[UI] fallback: ${textureKey}\`)`。

### 2. UIAvatar 纹理缺失时未执行统一 fallback war

- **位置**: `apps/phaser3/src/ui/UIAvatar.ts:89-104`
- **问题**: `createContent()` 中当 `texture` 非空但 `hasTexture` 返回 false 时，直接回退到 `Phaser.GameObjects.Text` 渲染，未打印 fallback warn。
- **影响**: 同问题 1，违反资源回退契约。且 Text 回退的视觉效果（文字居中）与组件预期（图片头像）差异较大，无 warn 难以排查。
- **修复**: 在 `else` 分支前增加 `console.warn(\`[UI] fallback: ${texture}\`)`，或改用 `createFallbackRect` 作为视觉回退（保留 fallbackText 为次要方案）。
- **依据**: PRD-004 3.3 节资源回退统一标准。

### 3. UIResourceDisplay 空 icon 字符串触发无意义 fallback war

- **位置**: `apps/phaser3/src/ui/UIResourceDisplay.ts:76-83`
- **问题**: `createContent()` 中 `if (hasIcon)` 为 true 时（`icon` 默认为空字符串 `''`），`hasTexture(this.scene, '')` 返回 false，进入 `createFallbackRect(this.scene, ..., '', ...)`，打印 `[UI] fallback: `。
- **影响**: 空字符串不应被视为有效的纹理键名，不应触发 warn，产生无意义的控制台噪音。
- **修复**: 在调用 `hasTexture` 前增加空字符串检查：
  ```typescript
  if (this.config.icon && hasTexture(this.scene, this.config.icon)) {
    // ...
  } else if (this.config.icon) {
    const fallback = createFallbackRect(...);
  }
  ```
- **依据**: PRD-004 3.3 节，fallback warn 应仅在"依赖纹理的组件纹理缺失"时触发；空字符串不属于有效的纹理依赖。

### 4. UIIconButton hover 态缺乏独立颜色变化，旧 API hoverColor 完全失效
- **位置**: `apps/phaser3/src/ui/UIIconButton.ts:125-145`
- **问题**: `pointerover` 时仅执行 `setFillStyle(this.config.bgColor, 1)`（透明度 0.9→1.0），没有使用 `hoverColor` 改变背景色。新接口已移除 `hoverColor` 的 Required 类型，但旧 API 兼容保留了 `width/height` 却未保留 `hoverColor`。结果 Hover 态与 Normal 态仅在透明度上有微弱差异，视觉上区分不明显。
- **影响**: 与 `ui-style-guide/03-component.md` 3.1 节 UIButton 三态标准不一致（Hover 背景应为 `hoverColor`）；旧代码迁移后 hover 视觉效果降级。
- **修复**: 恢复 `hoverColor` 的兼容支持，在 `pointerover` 时：
  ```typescript
  const hoverColor = (this.config as any).hoverColor;
  if (hoverColor) {
    this.background.setFillStyle(hoverColor, 1);
  } else {
    this.background.setFillStyle(this.config.bgColor, 1);
  }
  ```
或在重构时彻底移除 `bgColor/hoverColor` 语义，改为依赖 `textures` 三态切换。
- **依据**: `docs/design-docs/ui-style-guide/03-component.md` 3.1 节：Hover 态背景应为 `hoverColor`（透明度 1.0）。

### 5. UIList.scrollToIndex 为空实现
- **位置**: `apps/phaser3/src/ui/UIList.ts:206-208`
- **问题**: `scrollToIndex(_index: number)` 方法仅有注释占位，无任何实现逻辑。
- **影响**: PRD-004 5.7 明确将此方法列为 `UIList` 公共接口，空实现意味着接口契约未兑现。业务代码调用此方法无效果，可能导致开发者困惑。
- **修复**: 若当前确实无法接入 `UIScrollView`，应至少抛出 `console.warn('[UI] scrollToIndex is not yet implemented')` 或从类型定义中移除该方法，避免误导。
- **依据**: PRD-004 5.7 节接口设计：`scrollToIndex(index: number): void`。

### 6. UIIconButton 中 createFallbackRect 被强制类型转换为 Image
- **位置**: `apps/phaser3/src/ui/UIIconButton.ts:71-77` 及 `158-168`
- **问题**: `createFallbackRect` 返回 `Phaser.GameObjects.Rectangle`，但被 `as unknown as Phaser.GameObjects.Image` 强制转换后赋值给 `this.iconImage`。
- **影响**: 运行时不会崩溃（两者都有 setDisplaySize/setOrigin 等方法），但类型系统被欺骗，后续若调用 Image 特有的方法（如 `setFrame`）会在运行时报错。维护性差。
- **修复**: 将 `iconImage` 的类型声明为 `Phaser.GameObjects.Image | Phaser.GameObjects.Rectangle`，或使用一个统一的 `Phaser.GameObjects.GameObject` 类型。
- **依据**: TypeScript 最佳实践，禁止 `as unknown as` 类型欺骗。

### 7. 测试覆盖存在核心路径缺口
- **位置**: 多个测试文件
- **问题**:
  - `UIButton.test.ts` 未测试 `pointerup` 触发 `onClick`、`pointerupoutside` 取消 Pressed 态且不触发点击的新行为。
  - `UIIconButton.test.ts` 未测试 `textures` 三态切换（normal/hover/pressed/disabled）。
  - `UIPanel.test.ts` 未测试模态与非模态的 depth 差异（2000 vs 3000）。
  - 所有组件测试均未验证 `killTweensOf` 是否在动画前被调用（PRD-004 3.4 "动画铁律"）。
  - `UIAvatar.test.ts` 未测试 `isCircular` 遮罩在 `setAvatarSize` 后的正确重建。
- **影响**: 核心交互路径（按钮状态机、动画安全、depth 规范）缺乏自动化保护，回归风险高。
- **修复**: 补充上述测试用例。MockScene 已支持 `scene.tweens.tweens` 数组和 `emitEvent`，具备测试条件。
- **依据**: PRD-004 2.1 P1 验收标准："测试覆盖 show/hide、交互状态、资源回退"；`docs/design-docs/ui-style-guide/03-component.md` 3.1 节 "任何 tween 播放前必须先 killTweensOf"。

## 轻微问题（Minor）

### 1. TopInfoBar 仍保留魔法数字色值
- **位置**: `apps/phaser3/src/city-builder/ui/TopInfoBar.ts:162-178`
- **问题**: `bgColor: 0xffffff`、`strokeColor: 0xc9a227` 等硬编码色值未替换为 `UITheme` 常量。
- **影响**: 与国风色板规范存在偏差，后期统一修改困难。
- **修复**: 将 `0xc9a227` 替换为 `UITheme.goldBrown`，`0xffffff` 替换为 `UITheme.white`。
- **依据**: `docs/frontend/agent-design-guide.md` Step 3 检查清单："颜色使用 UITheme 常量，禁止魔法数字"。

### 2. UIButton pressedColor 使用哨兵值 -1
- **位置**: `apps/phaser3/src/ui/UIButton.ts:42`
- **问题**: `pressedColor: config.pressedColor ?? -1` 使用 `-1` 作为未配置的哨兵值，在 `pointerdown` 中通过 `!== -1` 判断。
- **影响**: 不够优雅；若用户传入 `-1` 作为有效色值会被误判为未配置。
- **修复**: 使用 `undefined` 作为默认值：`pressedColor?: number`，判断时改为 `if (this.config.pressedColor !== undefined)`。
- **依据**: 代码可读性最佳实践。

### 3. UIAlert.hide 使用 UITweens.toastOut 语义不清
- **位置**: `apps/phaser3/src/ui/UIAlert.ts:137-148`
- **问题**: `UIAlert.hide()` 使用了 `UITweens.toastOut.duration` 和 `UITweens.fade.ease`。虽然数值正确（200ms / Power2），但语义上 UIAlert 并非 Toast，混用 `toastOut` 配置容易造成维护困惑。
- **修复**: 统一使用 `UITweens.fade`（150ms）或新增 `UITweens.alertOut`（200ms），保持配置语义与组件对应。
- **依据**: 代码可读性。

### 4. docs/modules/client/ui.md 与 PRD-004 接口不一致
- **位置**: `docs/modules/client/ui.md:96-103`
- **问题**: 文档将 `UIIconButtonConfig.textures` 标记为可选（`textures?:`），但 PRD-004 5.4 中 `textures` 为必填字段。
- **影响**: 文档与需求源（PRD）存在偏差，开发者可能产生困惑。
- **修复**: 统一文档与 PRD 的接口定义；若实现上确实提供了默认值 `{ normal: '' }`，可在 PRD 中标注为可选，或保持 PRD 权威来源不变并在文档中同步。
- **依据**: PRD-004 5.4 节接口设计。

### 5. UIListItem 因 Phaser 内置方法冲突未使用 PRD 规定的 setData/getData
- **位置**: `apps/phaser3/src/ui/UIListItem.ts:130-136`
- **问题**: PRD-004 5.7 规定接口为 `setData(data: unknown): void` / `getData(): unknown`，但实现使用 `setItemData` / `getItemData`，因为 `Phaser.GameObjects.GameObject` 已内置 `setData` 方法。
- **影响**: 接口与 PRD 不一致，但属于合理的技术约束。
- **修复**: 在 PRD-004 中更新接口名称为 `setItemData` / `getItemData`，或在 UIListItem 文档中明确说明冲突原因。
- **依据**: Phaser3 API 文档，`GameObject.setData(key, value)` 为内置方法。

### 6. UIPanel 关闭按钮未复用 UIIcon 组件
- **位置**: `apps/phaser3/src/ui/UIPanel.ts:141-186`
- **问题**: 设计规范 issue-19 验收标准 9.1 要求 "关闭按钮使用 UIIcon + cancel-btn-0.png"，但实现中直接手动创建 `Phaser.GameObjects.Image` / `createFallbackRect` 并封装到 Container 中。
- **影响**: 与验收标准存在偏差；未复用现有组件，增加维护成本。
- **修复**: 将关闭按钮替换为 `new UIIcon(this.scene, { texture: 'cancel-btn-0', size: 24, isInteractive: true })`，并绑定事件。
- **依据**: `designs/issue-19/design-spec.md` 9.1 节验收标准。

## 风险与建议

### 短期风险
1. **资源回退 warn 不一致**：UIPanel 和 UIAvatar 在纹理缺失时静默回退，可能导致线上环境中美术资源漏加载难以被发现。建议在所有依赖纹理的组件中统一使用 `createFallbackRect` 进行缺失纹理回退。
2. **按钮状态机测试缺失**：UIButton 的 `pointerup` / `pointerupoutside` 分离是新引入的行为（将 onClick 从 pointerdown yoyo 动画完成时触发改为 pointerup 时触发），测试未覆盖。建议补充测试以防止后续重构破坏该行为。

### 中期建议
1. **PRD-004 更新**：PRD 仍为 Draft 状态，且部分接口（如 UIListItem.setData）与 Phaser 内置方法冲突。建议同步更新 PRD，标记 `setItemData` 为实际接口，并调整 Layer 2 交付计划。
2. **设计资产同步验证**：本次 PR 修改了大量 `designs/issue-19/src/ui/` 中的设计交付物，但 PR 说明强调 "实现以 `apps/phaser3/src/ui/` 为准"。建议确认 `designs/issue-19/` 中的代码与 `apps/phaser3/src/ui/` 的差异是否已收敛，避免设计交付物与最终实现漂移。
3. **空 icon 的默认行为**：UIResourceDisplay、UIListItem、UIIconButton 等多个组件在 `icon` 为空字符串时行为不一致（有的触发 fallback warn，有的跳过）。建议在 utils 中增加 `hasValidTexture(scene, key)` 工具函数，统一处理空字符串情况。
