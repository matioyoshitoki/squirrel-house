- **依据**: `docs/frontend/agent-design-guide.md` 3.1 节要求 `designs/issue-xxx/src/` 目录结构与 `apps/phaser3/src/` 对齐；Step 4 检查清单要求"所有 `src/` 代码在 `designs/` 目录内可编译（无导入错误）

- **位置**: 
  - `designs/issue-19/src/ui/UIToast.ts:155-163`（show 动画硬编码 `duration: 300, ease: 'Back.out'`）
  - `designs/issue-19/src/ui/UIToast.ts:180-191`（dismiss 动画硬编码 `duration: 200, ease: 'Power2'`）
  - `designs/issue-19/src/ui/UIAlert.ts:117-125`（show 动画硬编码 `duration: 200, ease: 'Back.out'`）
  - `designs/issue-19/src/ui/UIAlert.ts:135-151`（hide 动画硬编码 `duration: 200, ease: 'Power2'`）

### 13. `ToastManager.clear()` 直接调用 `destroy()`，缺少退场动画
- **位置**: `designs/issue-19/src/ui/UIToast.ts:264-267`
- **问题**: `clear()` 对每个 Toast 直接 `destroy()`，未走 `dismiss()` 的退场动画流程。虽然语义上可能是"立即清空"，但与组件动画规范不一致。
- **修复**: 若预期为立即强制清理，可在代码注释中注明 `// 强制立即销毁，不播放退场动画`；若预期为优雅清理，改为调用 `toast.dismiss()`。

### 14. `UIAlert.ts` 使用类型色作为整体边框色
- **位置**: `designs/issue-19/src/ui/UIAlert.ts:79`
- **问题**: `this.background.setStrokeStyle(2, color);` 使用 `type` 对应的类型色作为外边框。而 PRD-004 / design-spec 中 UIAlert 的视觉标准仅要求"左侧 4px 色条"作为类型标识，未要求整体边框变色。UIPanel 的规范边框是 `UITheme.goldBrown`。
- **修复**: 将边框色统一为 `UITheme.goldBrown`，或仅在 design-spec 中明确补充"整体边框使用类型色"的设计决策。

## 风险与建议

1. **编译阻断风险最高**：`designs/issue-19/src/ui/` 目录下缺少 `utils.ts`、`types.ts`、`UIButton.ts`，导致整个设计交付物无法编译。建议开发 Agent 优先补齐这三个文件，再执行 `pnpm --filter phaser3 type-check` 验证。

2. **迁移接口冲突**：`UISegmentedControl` 的接口从 `selectedId` / `onSelect` 改为 `selectedIndex` / `onChange`，`SceneSwitchBar` 已适配，但需确认其他引用 `UISegmentedControl` 的旧业务代码是否已全部迁移（本次 diff 仅看到 `SceneSwitchBar.ts` 一处修改）。

3. **Phaser 基类方法覆盖**：`UIBadge.setSize` 更名为 `setBadgeSize` 是必要的技术决策（规避 `Container.setSize`），但请在 PRD / 接口文档中同步更新，避免 Developer Agent 迁移时按旧接口调用导致编译错误。

4. **建议补充截图验收**：根据 `agent-design-guide.md` Step 5，新组件应产出 Normal / Hover / Pressed / Disabled / page-context 截图，存入 `designs/issue-19/screenshots/`。
