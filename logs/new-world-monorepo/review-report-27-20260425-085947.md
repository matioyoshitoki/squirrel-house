## 风险与建议

1. **迁移路径风险**: `index.ts` 中 Layer 1 / Layer 2 导出全部被注释。Developer Agent 迁移时需按组件就绪情况逐步取消注释，避免一次性引入大量未实现的导入导致编译失败。建议在迁移指南中增加 "分组件逐条取消注释" 的提示。

2. **`feature/issue-18-ui-infrastructure` 分支冲突**: `design-spec.md` 已明确指出该分支产物为 React + Tailwind 方向，与项目 Phaser3 技术栈严重不符。建议仓库维护者考虑删除或重命名 `feature/issue-18-ui-infrastructure` 分支，防止后续 Agent 误引用。

3. **`getNextDepth` WeakMap 泄漏风险**: 当前实现使用 `WeakMap<Scene, Map<...>>`，依赖 Scene GC 自动回收。测试环境中若 MockScene 未正确释放，`depthCounters` 会持续累加。建议在 `UIMockScene` 的测试使用说明中提示 "每个 test 创建新的 UIMockScene 实例"。

4. **`setup.ts` 硬编码路径**: 迁移时需将 `../../../../../apps/phaser3/src/test-setup` 替换为 `'../../test-setup'`。当前代码和文档已对此有明确注释，但仍属于易错点。建议提供一个迁移脚本或 `sed` 命令在 `design-spec.md` 中。
