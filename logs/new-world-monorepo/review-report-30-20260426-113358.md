- **依据**:

  - `designs/issue-19/design-spec.md` 1.1 节明确声明单向依赖 `UIDialog → UIPanel → utils.ts / types.ts`，不得引用依赖中不存在的符号

### 2. UIDialogConfig 未按 PRD-004 继承 UIPanelConfig
- **位置**: `designs/issue-19/src/ui/UIDialog.ts:16`
- **问题**: 
  - PRD-004 5.6 明确要求 `export interface UIDialogConfig extends Omit<UIPanelConfig, 'title' | 'showClose'>`
  - 当前实现为 `export interface UIDialogConfig extends UIPositionConfig, Partial<UISizeConfig>`，**未继承** `UIPanelConfig`
  - 导致 `UIDialog` 无法通过配置传入 `showOverlay`、`overlayAlpha`、`isModal`、`onClose`、`onOverlayClick`、`bgColor`、`strokeColor`、`texture` 等 `UIPanel` 已支持的属性
- **影响**: 配置灵活性受限，与 PRD-004 定义的标准接口不兼容，Developer Agent 迁移后若业务需要自定义 Dialog 背景/遮罩行为，必须回头修改接口
- **修复**:
  ```typescript
  // designs/issue-19/src/ui/UIDialog.ts
  export interface UIDialogConfig extends Omit<UIPanelConfig, 'title' | 'showClose'> {
    title: string;
    message: string;
    // ... 保留其余字段
  }
  ```
- **依据**: PRD-004 第 5.6 节接口设计代码块明确声明 `extends Omit<UIPanelConfig, 'title' | 'showClose'>`

### 4. 硬编码色值违反 Foundation 规范 F-07
- **位置**: 
  - `designs/issue-19/src/ui/UIAlert.ts:113`：`color: '#ffffff'`
  - `designs/issue-19/src/ui/UIDialog.ts:104`：`color: '#cccccc'`
- **问题**: 
  - 两处文字样式均直接手写对象，未使用 `getInkTextStyle()`，且使用了硬编码 HEX 色值
  - `01-foundation.md` F-07 明确禁止「在业务代码中直接写魔法数字色值」
  - `01-foundation.md` 1.2 节指出「通过 `getInkTextStyle()` 自动注入，不需要手动写」
  - 尤其 `#cccccc`（浅灰）不在 `UITheme` 常量中，属于纯魔法数字
- **影响**: 风格不一致，维护困难；后续统一调整色板时无法通过修改 `UITheme` 常量批量生效
- **修复**:
  ```typescript
  // designs/issue-19/src/ui/UIAlert.ts
  this.messageText = this.scene.add.text(
    0, 0, message,
    getInkTextStyle('14px', colorToHex(UITheme.white))
  ).setOrigin(0.5);

  // designs/issue-19/src/ui/UIDialog.ts
  this.messageText = this.scene.add.text(
    0, -10, message,
    getInkTextStyle('14px', colorToHex(UITheme.lightInk), {
      align: 'center',
      wordWrap: { width: width - 40 },
    })
  ).setOrigin(0.5);
  ```
  > 注：`#cccccc` 与 `UITheme.lightInk`（`#666666`）差异较大。若设计 spec 确实需要浅灰，应在 `UITheme` 中新增常量（如 `subText: 0xcccccc`），否则建议改用 `UITheme.lightInk` 以符合色板规范。
- **依据**: 
  - `docs/design-docs/ui-style-guide/01-foundation.md` 1.2 节字体规范
  - `docs/design-docs/ui-style-guide/01-foundation.md` F-07 禁止清单

- **位置**: `designs/issue-19/src/ui/UIToast.ts:257-261`
- **问题**: `toast['dismiss'] = () => { ... }` 绕过 TypeScript `private` 访问控制，属于类型系统 hack，长期维护易引发问题

### 7. UIDialog.confirm/cancel 中 killTweensOf 冗余
- **位置**: `designs/issue-19/src/ui/UIDialog.ts:149`、`designs/issue-19/src/ui/UIDialog.ts:157`
- **问题**: `confirm()` / `cancel()` 已调用 `killTweensOf(this.scene, this)`，而 `hide()` 内部也会再次调用，存在冗余
- **修复**: 移除 `confirm()` / `cancel()` 中的 `killTweensOf`，由 `hide()` 统一处理（防御性保留亦可，非强制）

### 8. Warning 类型色与 PRD-004 5.15 存在项目级文档冲突
- **位置**: `designs/issue-19/src/ui/UIAlert.ts:67`、`designs/issue-19/src/ui/UIToast.ts:74`、`designs/issue-19/design-spec.md:231/310`

  - `docs/design-docs/ui-style-guide/03-component.md` 3.7 节同样使用 `goldBrown`
- **说明**: 这是项目内部 **PRD 与组件规范之间的冲突**，并非代码单方面错误。建议产品侧确认以哪个为准，并同步更新冲突文档
- **修复（二选一）**: 
  - 方案 A：代码改用 `UITheme.warning`，同步更新 `03-component.md`
  - 方案 B：保持 `UITheme.goldBrown`，在 PRD-004 中修正 warning 色说明
