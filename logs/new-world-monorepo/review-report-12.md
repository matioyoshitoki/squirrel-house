### 🟡 问题 2: 排行榜查询存在 N+1 数据库查询风险
- **文件**: `internal/data/wapi/leaderboard.go:64-104`
- **描述**: `GetLeaderboard` 在遍历 Redis 返回的每个成员时，都会调用 `loadUsername` 查询一次 `worldDB.User`，导致每页请求产生 `pageSize` 次数据库查询。当 `DisplayCount` 较大（如 100）时，可能对数据库造成压力。
- **建议**: 考虑以下优化方向之一：
  1. 将用户名与用户 Code 一并缓存到 Redis 成员值中（如 JSON 序列化存储）；
  2. 使用 `IN` 批量查询用户名称，减少查询次数；
  3. 增加本地缓存（LRU）或单独的用户信息 Redis 缓存层。
